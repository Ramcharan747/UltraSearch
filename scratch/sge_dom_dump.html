To read a file line by line in Go, the most idiomatic and memory-efficient way is to use the bufio.Scanner. This approach handles the buffering for you, ensuring that you don't load the entire file into memory at once. 
Using bufio.Scanner (Recommended) 
This method is preferred because it's clean and handles the "newline" splitting automatically. 
go
package main

import (
	"bufio"
	"fmt"
	"log"
	"os"
)

func main() {
	// 1. Open the file
	file, err := os.Open("example.txt")
	if err != nil {
		log.Fatal(err)
	}
	// 2. Ensure the file is closed when the function finishes
	defer file.Close()

	// 3. Create a scanner to read the file
	scanner := bufio.NewScanner(file)

	// 4. Iterate through each line
	for scanner.Scan() {
		// scanner.Text() returns the line without the newline character
		fmt.Println(scanner.Text())
	}

	// 5. Check for errors that occurred during scanning
	if err := scanner.Err(); err != nil {
		log.Fatal(err)
	}
}
Alternative Methods 
While the scanner is the standard choice, you may need other approaches depending on your specific requirements: 
  • For very long lines: The default bufio.Scanner has a maximum token size limit of 64KB. If your file contains lines longer than this, you must either increase the buffer using scanner.Buffer() or use bufio.Reader with ReadString('\n'). You can find discussions on handling long lines in this Stack Overflow thread. 
  • For CSV/TSV data: If the file is structured as a spreadsheet, the standard library's encoding/csv package is more appropriate as it handles delimiters and quotes automatically. 
  • Using bufio.Reader: For more granular control, Practical Go Lessons suggests using bufio.Reader for cases where you need to keep the delimiter or handle EOF (End of File) conditions manually. 
  • Small Files: For very small files where memory is not a concern, you can read the entire content into a slice and split it, as detailed in this Medium article on reading the right way.
